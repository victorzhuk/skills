# Go family gap table — skills vs real code

Working audit doc, not for public release. Every row compares a `skills/go/z-go-*/SKILL.md`
claim against what `~/Projects/own/` repos actually do, per
`~/.agents/references/VICTOR_ENGINEERING_BASELINE.md`. Generated by 6 parallel
audit passes (batches of ~5-6 skills each); two batches needed a retry after
transient API errors.

Legend: **drift** = skill states X as the default/equal option, real code never
uses it, nothing suggests deliberate future intent. **aspiration** = a
capability real code hasn't exercised, but something (a private rule, an
in-flight modernization doc, a design note) suggests deliberate intent —
flagged, not resolved. **ambiguous** = genuinely unclear, needs a call.

## Correction to the baseline doc

`VICTOR_ENGINEERING_BASELINE.md` currently states testify as the universal Go
testing default. Evidence from this audit shows it's the **majority** pattern,
not universal — the flagship exemplar `gdebenz-buddy-bot` itself is
stdlib-only (`testing` + hand-rolled fakes + `goleak`, zero testify). Needs a
wording fix before Phase 3 edits use it as ground truth.
