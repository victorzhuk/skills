# Engineering Skills

General engineering process, changelog, architecture, repository tooling, and skill authoring.

## Model-invoked

Model- or user-reachable via skill name and trigger phrasing.

- **[z-changelog](./z-changelog/SKILL.md)** — Maintain a Keep a Changelog 1.1.0 CHANGELOG.md — [Unreleased] upkeep, the six change types, the release cut, YANKED, the SemVer note, and git-cliff/goreleaser coexistence.
- **[z-clean-output](./z-clean-output/SKILL.md)** — Keep code, docs, comments, configs, and commits free of AI/generation/process references — no "Generated by", no rule-name citations, no meta-commentary about how the output was produced.
- **[z-codegraph](./z-codegraph/SKILL.md)** — Use when working with CodeGraph code intelligence: symbol search, call graph exploration, impact analysis, affected tests, route-to-handler lookup, or codebase understanding in repositori…
- **[z-creating-flow-doc](./z-creating-flow-doc/SKILL.md)** — Use when adding a new flow document to a behavioural-contract docs directory (e.g. `docs/{domain}/flows/`), or documenting an operation, RPC, cron, or query as a single-page paired-diagra…
- **[z-domain-modeling](./z-domain-modeling/SKILL.md)** — Build and maintain a domain model — ubiquitous language, bounded contexts, aggregates and invariants, and deep vs shallow modules.
- **[z-memory-hygiene](./z-memory-hygiene/SKILL.md)** — Use when storing, updating, or reviewing long-term agent memory. Covers what belongs in memory, category/scope choices, importance, verification by recall, and safe self-improvement notes…
- **[z-no-ai-style-code](./z-no-ai-style-code/SKILL.md)** — Write like a senior maintainer of this specific project, not like a tutorial — natural short names, concise error context, structure that matches the surrounding code's idiom.
- **[z-no-over-engineering](./z-no-over-engineering/SKILL.md)** — YAGNI discipline for the smallest correct change — no interfaces "for testing", no DTOs when a domain struct suffices, no premature package splits or configuration layers.
- **[z-npm-publish](./z-npm-publish/SKILL.md)** — Publish a Node/TypeScript package to npm via GitHub Actions with OIDC trusted publishing and provenance.
- **[z-skill-mastery](./z-skill-mastery/SKILL.md)** — Author, refactor, and review agent skills in this repo's house style.
- **[z-testing-strategy](./z-testing-strategy/SKILL.md)** — Choose test depth and behavior-contract approach from the type of work, not a fixed coverage target — MVP/greenfield TDD vs existing-service regression protection.
